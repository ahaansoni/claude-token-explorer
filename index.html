The history trap (affects Pro users too): Every new message re-reads your full conversation history. Message 20 sends all 19 previous messages + your new one as input every time. This is why Pro users hit daily limits after long sessions.
8 tips to save tokens
Works for both API users and Claude Pro $20/month subscribers.
1Ask for shorter output explicitly
"In 2 sentences", "3 bullets only" output costs 5× more than input. Biggest single saving.
2Use Haiku for simple tasks
Summarizing, classifying, formatting? Haiku 4.5 is 3× cheaper than Sonnet. Save Sonnet for complex reasoning.
3Start a new chat for new topics
Every reply re-reads full conversation history. New topic = fresh chat. Saves tokens and resets your Pro daily weight.
4Keep your system prompt short
Sent with every single message. Cutting one sentence saves thousands of tokens over a session.
5Batch your questions
One message with 5 questions is far cheaper than 5 separate messages. Less history compounding.
6Paste only what's needed
Uploading a 50-page doc when you need page 3? Paste just the relevant section.
7Use structured prompts
Format: Context: [...]. Task: [...]. Format: [...]. Less ambiguity = fewer back-and-forth turns.
8Skip the pleasantries
"Please", "thank you", "could you kindly" adds tokens with zero benefit to output quality.

Why does my Pro session drain so fast?
Limits are compute-weighted not message-count. One message with a 50-page PDF + Opus model + Research tool enabled can drain your 5-hour session faster than 20 short Haiku messages. Long conversation history also compounds on every turn.
For Claude Code users
Using Claude Code (CLI)? You have extra tools to manage tokens that regular chat users don't.
In-session commands
/clear
Wipes full conversation history. Start fresh project files and CLAUDE.md stay. Use when switching tasks. Cannot be undone.
/compact
Summarizes conversation history into a short recap, freeing context while keeping essential context. Use mid-task when you need to keep going.
/cost
Shows running spend for the current session (API key users only). Track how much you've burned before switching models or tasks.
/model
Switch models mid-session without losing conversation. Common pattern: plan with Opus, execute with Sonnet.
ccusage rack Claude Code usage from terminal
npx ccusage reads your local Claude Code session files and generates daily, weekly, and per-session reports. No API key needed. Especially useful for Pro/Max users since the Console doesn't show per-token data for flat-rate plans.

npx ccusage blocks shows usage by 5-hour billing window
npx ccusage daily --breakdown per-model cost breakdown
